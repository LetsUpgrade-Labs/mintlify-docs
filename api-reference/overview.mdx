## Base URL

```
https://nexi.thelisa.ai
```

## Authentication

Most endpoints require authentication. Both methods work on **every** endpoint -- use whichever suits your use case. See the [Authentication guide](/guides/authentication) for full details.

| Method | How | Best for |
|--------|-----|----------|
| JWT Cookie (`auth-token`) | Auto-set after `/api/auth/login` or `/api/auth/google` | Browser / Web UI |
| API Key | `Authorization: Bearer sk-xxx` header | curl, Postman, scripts, server-to-server |
| Admin | Either method, requires `admin` role | `/api/admin/*` endpoints |

<Tip>
  For **curl or programmatic access**, use `Authorization: Bearer sk-xxx`. No login step needed.
</Tip>

## Provider Selection

The `/api/generate-video` endpoint accepts optional provider parameters to customize which AI models are used for each stage of the pipeline:

| Parameter | Type | Description | Example |
|-----------|------|-------------|---------|
| `llmProvider` | string | LLM for script generation (`provider:model`) | `anthropic:claude-sonnet-4-20250514` |
| `ttsProvider` | string | TTS engine for voiceover (`provider:model`) | `openai:tts-1-hd` |
| `imageProvider` | string | Image generation model (`provider:model`) | `kie:nano-banana-pro` |
| `imageStyle` | string | Visual style for scene images | `doodle`, `animated-cartoon`, `photorealistic` |
| `voiceId` | string | Provider-specific voice identifier | `nova`, `meera`, or an ElevenLabs voice ID |

All fields are optional. Sensible defaults are used when omitted. The default TTS is `openai:tts-1` with voice `alloy`.

<Warning>
  If you provide a `voiceId`, it **must** be valid for the chosen `ttsProvider`. Invalid voice IDs are rejected with a `400` error before the pipeline starts.
</Warning>

### LLM Providers Reference

The `llmProvider` parameter controls which language model generates the video script and scene structure.

| Model ID | Label | Provider |
|----------|-------|----------|
| `anthropic:claude-sonnet-4-20250514` | Claude Sonnet 4 (default) | Anthropic |
| `anthropic:claude-haiku-4-20250414` | Claude Haiku 4 | Anthropic |
| `openai:gpt-4o` | GPT-4o | OpenAI |
| `openai:gpt-4o-mini` | GPT-4o Mini | OpenAI |
| `google:gemini-3.1-flash-lite-preview` | Gemini 3.1 Flash Lite (preview) | Google |

```json
{ "llmProvider": "openai:gpt-4o" }
```

### Image Descriptions

You can pass `userImageDescriptions` alongside `userImageUrls` to help the AI use your uploaded images more effectively:

```json
{
  "userImageUrls": ["https://example.com/logo.png", "https://example.com/product.jpg"],
  "userImageDescriptions": [
    { "url": "https://example.com/logo.png", "description": "Company logo", "isLogo": true },
    { "url": "https://example.com/product.jpg", "description": "Product screenshot" }
  ]
}
```

## Rate Limits

Rate limiting is enforced at **10 requests per minute per API key**. The following headers are included in every response:

| Header | Description |
|--------|-------------|
| `X-RateLimit-Limit` | Maximum requests allowed per minute |
| `X-RateLimit-Remaining` | Requests remaining in the current window |
| `X-RateLimit-Reset` | Unix timestamp when the window resets |

Exceeding the limit returns a `429 Too Many Requests` response.

## Errors

All error responses follow this format:

```json
{
  "error": "Human-readable error message"
}
```

The `/api/generate-video` endpoint returns additional diagnostic fields on failure:

```json
{
  "error": "Failed to start video generation",
  "step": "trigger_task",
  "detail": "TRIGGER_SECRET_KEY is not configured"
}
```

For **async** pipeline failures (TTS errors, rendering errors, etc.), poll `/api/video-status` -- the `error` field contains the specific failure message:

```json
{
  "status": "FAILED",
  "videoId": "...",
  "error": "openai TTS failed (model: openai:tts-1, voice: \"invalid\"): 400 Bad Request"
}
```

Common HTTP status codes:

| Code | Meaning |
|------|---------|
| `200` | Success |
| `201` | Created |
| `202` | Accepted (async task started) |
| `400` | Bad request (missing/invalid params, invalid voice ID) |
| `401` | Not authenticated |
| `403` | Forbidden (wrong role or owner) |
| `404` | Resource not found |
| `429` | Rate limit exceeded |
| `500` | Server error |
